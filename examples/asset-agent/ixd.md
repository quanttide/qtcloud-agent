# 资产智能体协作壳 — 页面原型设计文档

文档状态：v1.0 草案
角色：产品经理
日期：2026-05-16

—

1. 产品背景与目标

背景
平台资产库以树状结构组织，各级目录驻留智能体，通过文件变化触发协作。目前协作流程依靠人工模拟，缺乏直观的监控与理解工具。

目标
设计一个本地运行的“壳”界面，提供以下核心能力：

· 可视化资产树与智能体的职责分布
· 实时展示智能体间的协作链路与触发关系
· 回溯历史事件，理解协作的完整过程
· 支持手动介入，发送模拟事件或调整规则

用户
当前仅创始人本人，需零配置启动，开箱即用。

—

2. 设计原则

· 所见即所得：界面直接映射资产目录结构，无额外抽象层。
· 图为中心：协作关系用有向图呈现，节点可交互。
· 事件透明：每次触发和反馈均以标准化事件卡片展示。
· 零学习成本：无需文档，看树、看图即能理解系统状态。

—

3. 整体布局

```
┌──────────────────────────────────────────────────────────┐
│  顶部工具栏（切换视图、手动触发、设置）                    │
├────────────┬──────────────────────────┬───────────────────┤
│            │                          │                   │
│  资产树    │      协作关系图          │   事件时间线      │
│  (左面板)  │      (中央主区域)        │   (右面板)        │
│            │                          │                   │
│  展示目录  │  智能体节点 + 资产节点   │  按时间倒序的    │
│  树，挂载  │  有向边表示触发/反馈    │  事件卡片列表    │
│  智能体    │  支持拖拽、缩放、高亮   │  可展开查看报文  │
│            │                          │                   │
├────────────┴──────────────────────────┴───────────────────┤
│  底部状态栏（引擎状态、最近事件摘要、连接状态）            │
└──────────────────────────────────────────────────────────┘
```

比例建议：左面板 250px 固定，右面板 300px 固定，中间自适应。

4. 组件详细设计

4.1 资产树（左面板）

功能：展示仓库目录结构，标注智能体挂载位置。

视觉元素：

· 树形控件，文件夹/文件图标
· 目录节点右侧显示已绑定的智能体标签（如 🤖 drd-agent）
· 当前活跃的智能体用绿色圆点标记
· 有事件发生的目录高亮闪烁（持续 3 秒）

交互：

· 点击目录节点 → 中央图中对应节点居中放大
· 悬停节点 → 显示智能体详情（监控路径、最近触发时间）
· 右键菜单：手动触发该智能体、查看规则、打开目录

数据源：静态资产目录 + 智能体配置（agent_rules.yaml）

4.2 协作关系图（中央主区域）

功能：以有向图展示智能体与资产的触发关系。

节点类型：

· 智能体节点（圆角矩形，带图标）：显示智能体 ID
· 资产节点（文档图标）：关键文件（如 PLAN.md、STATUS.md）
· 事件节点（菱形，仅在图链路中短暂出现）：当前传递的事件

边的样式：

· 正向触发：实线箭头，标签显示事件类型（如 drd.finalized）
· 反向反馈：虚线箭头，橙色，标签显示 change.request
· 当前活跃边：高亮动画（流动粒子），表示正在传递的事件

交互：

· 拖拽节点调整布局
· 缩放（鼠标滚轮/触控板）
· 点击节点 → 高亮其所有入边和出边，其他边半透明
· 双击智能体节点 → 打开详情面板（规则、最近日志）
· 悬停边 → 显示触发条件和时间

自动布局：初始使用树形层次布局（从上到下：元仓库→单仓→分层），后续可手动调整并保存位置。

图例：置于图右下角，解释节点形状和线型含义。

4.3 事件时间线（右面板）

功能：按时间倒序列出所有协作事件，提供详细报文查看。

事件卡片结构：

```
┌─────────────────────────────────┐
│ 14:30:05  drd.finalized         │
│ 源: /平台/docs/drd/支付设计.md  │
│ 目标: qa-agent, code-agent      │
│ [展开报文 ▼]                    │
└─────────────────────────────────┘
```

展开后显示完整事件 JSON（CloudEvents 格式）。

交互：

· 点击事件卡片 → 图中对应边高亮，关联节点居中
· 悬停 → 显示简单 tooltip（谁触发谁）
· 支持筛选：按智能体、事件类型、时间范围
· 顶部显示事件统计（如“今日 12 个事件，3 个反馈”）

实时更新：新事件到达时，卡片从顶部滑入，图中同步动画。

4.4 手动触发面板（模态框）

入口：顶部工具栏按钮，或资产树右键菜单。

功能：

· 选择目标智能体
· 输入模拟的文件路径或事件数据
· 发送后模拟一次触发，观察图中链路反应

用途：验证规则、测试协作路径。

4.5 智能体详情抽屉（侧边滑出）

触发：双击图中智能体节点。

内容：

· 基本信息：ID、监控路径、命令
· 近期事件列表（该智能体相关）
· 出/入边列表（上下游智能体）
· 规则配置摘要（只读）

5. 关键使用场景

场景 1：查看当前分工

1. 打开壳 → 自动加载资产树和关系图
2. 浏览树：看到每个目录绑定的智能体
3. 图中直观显示整个协作拓扑

场景 2：追踪一次协作

1. 发现中央图出现流动高亮边
2. 点击边 → 右侧事件卡片定位并展开报文
3. 观察整条链路的节点依次激活

场景 3：调试反馈环

1. 看到虚线反向边从 QA 指向 DRD
2. 点击边 → 卡片显示 design.change.request 事件
3. 检查 data.suggestion 字段，验证报文内容

场景 4：模拟变更

1. 右键 DRD 目录 → “手动触发”
2. 输入“状态改为 final”
3. 图上立刻出现正向链路动画，验证下游是否按预期激活

6. 技术实现建议（前端视角）

· 框架：React + TypeScript（如有经验）或 vanilla HTML + vis.js，快速出原型
· 图库：React Flow 或 vis-network，内置有向图、动画、事件处理
· 树组件：自研简单递归组件，或使用第三方 Tree 组件
· 数据接口：初期使用静态 JSON 文件模拟；后期通过 WebSocket 从规则引擎推送实时事件
· 打包：Electron 或本地 Web 服务（如 python -m http.server），无需复杂构建

—

7. 原型验证计划

1. 第 1 天：用静态数据实现上述布局和核心图交互（资产树 + 关系图 + 假事件流）
2. 第 2 天：补充关键交互（高亮链路、事件卡片展开、手动触发模拟）
3. 第 3 天：创始人亲自体验，验证分工是否清晰、协作关系是否可理解
4. 反馈后：调整节点布局和交互细节，确认后端需提供的数据 API 格式
